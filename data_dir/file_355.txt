Creating a project¬ґ
If this is your first time using Django, youвАЩll have to take care of some
initial setup. Namely, youвАЩll need to auto-generate some code that establishes
a Django project вАУ a collection of settings for an instance of Django,
including database configuration, Django-specific options and
application-specific settings.
From the command line, cd into a directory where youвАЩd like to store your
code and run the following command to bootstrap a new Django project:


пЕЉ/пЕє

пЕЇ




...\> django-admin startproject mysite djangotutorial



This will create a directory djangotutorial with a project called
mysite inside. The directory name doesnвАЩt matter to Django; you can rename
it to anything you like. If it didnвАЩt work, see
Problems running django-admin.

Note
YouвАЩll need to avoid naming projects after built-in Python or Django
components. In particular, this means you should avoid using names like
django (which will conflict with Django itself) or test (which
conflicts with a built-in Python package).

LetвАЩs look at what startproject created:

These files are:

manage.py: A command-line utility that lets you interact with this
Django project in various ways. You can read all the details about
manage.py in django-admin and manage.py.
mysite/: A directory that is the actual Python package for your
project. Its name is the Python package name youвАЩll need to use to import
anything inside it (e.g. mysite.urls).
mysite/__init__.py: An empty file that tells Python that this
directory should be considered a Python package. If youвАЩre a Python beginner,
read more about packages in the official Python docs.
mysite/settings.py: Settings/configuration for this Django
project. Django settings will tell you all about how settings
work.
mysite/urls.py: The URL declarations for this Django project; a
вАЬtable of contentsвАЭ of your Django-powered site. You can read more about
URLs in URL dispatcher.
mysite/asgi.py: An entry-point for ASGI-compatible web servers to
serve your project. See How to deploy with ASGI for more details.
mysite/wsgi.py: An entry-point for WSGI-compatible web servers to
serve your project. See How to deploy with WSGI for more details.

application that does something вАУ e.g., a blog system, a database of

multiple apps. An app can be in multiple projects.
